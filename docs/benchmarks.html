Benchmarks

Rigorous evaluation of CortexDB against industry-standard RAG architectures across multiple multi-hop reasoning datasets.

Evaluation Overview

CortexDB was benchmarked against three baseline architectures — Naive RAG, HyDE (Hypothetical Document Embeddings), and Advanced RAG (with Re-ranker) — across datasets of varying complexity and size.

📊
MRR
Mean Reciprocal Rank — measures how high the first relevant result appears in the ranked list.
🎯
Ctx Precision
Context Precision — the fraction of retrieved contexts that are actually relevant to answering the query.
⏱️
Avg Latency
Average end-to-end latency measured in seconds for retrieving and generating an answer.
⭐
Answer Quality
LLM-evaluated answer correctness — the ultimate measure of how well the system answers multi-hop questions.
ℹ️ Evaluation Setup:

All benchmarks use 50 queries with a retrieval parameter of Top-K = 5 context elements. Latency is measured in seconds. MRR = Mean Reciprocal Rank; Ctx = Context.

HotpotQA-Derived Dataset (Small Size)

Evaluated using 50 queries mapped to approximately 120 context chunks from the HotpotQA dev-distractor split, including both gold evidence and semantic distractors.

Architecture MRR Ctx Precision Avg Latency Ans Quality
Naive 1.000 0.7 0.2486 0.7266
HyDE 0.9688 0.7 7.1957 0.7638
Advanced 1.000 0.7 7.7278 0.7824
CortexDB 1.000 0.7 2.3592 0.8242
